# best-hackaton

KISS mockup of a logistics decision-support pipeline driven by JSON contracts.

## Quick start

Run the full demo pipeline for a scenario:

```powershell
node src/cli/index.js run normal
node src/cli/index.js run demand_spike
node src/cli/index.js run blocked_route
```

Inspect available scenarios:

```powershell
node src/cli/index.js list-scenarios
```

Run modules one by one:

```powershell
node src/cli/index.js ai scenarios/normal/world_state.json pipeline/ai_output/ai_analysis.json
node src/cli/index.js prepare scenarios/normal/world_state.json pipeline/ai_output/ai_analysis.json pipeline/optimizer_input/optimizer_input.json
node src/cli/index.js optimize pipeline/optimizer_input/optimizer_input.json pipeline/solution/solution.json
node src/cli/index.js view scenarios/normal/world_state.json pipeline/solution/solution.json pipeline/ai_output/ai_analysis.json
```

Verify all scenarios and contracts:

```powershell
node src/cli/index.js verify
```

## Project map

- `contracts/` - canonical JSON examples and minimal required fields
- `docs/` - team rules and project guideline
- `mock/` - notes about mock module behavior and template outputs
- `pipeline/` - runtime artifacts generated by the CLI
- `scenarios/` - input scenarios for the demo
- `src/` - zero-dependency Node implementation of the mock pipeline

## Demo flow

```text
world_state.json
  -> ai_analysis.json
  -> optimizer_input.json
  -> solution.json
```

The modules communicate only through files, so each team member can work independently and replace a mock later with a real component.
